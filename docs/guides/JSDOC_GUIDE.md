## 1. When JSDoc Is Required vs. Optional

### Required

| Location | Rule |
|----------|------|
| Every exported function in `services/api/*.api.ts` | Required |
| Every exported hook in `hooks/*.ts` | Required |
| Every props interface in `components/*.tsx` | Required |
| Every Edge Function's `Deno.serve()` handler | Required |
| Non-obvious types/interfaces in `types/index.ts` | Required |

### Optional (only if non-obvious)

- Internal helper functions with self-documenting names
- Simple state variables (`const [isOpen, setIsOpen] = useState(false)`)
- Mapper functions that obviously transform A → B

### Never (these cases reduce signal-to-noise)

- **Auto-generated files** — `types/supabase.ts` is generated by Supabase CLI; comments will
  be overwritten on next `gen-db-types` run
- **Barrel re-exports** — `services/api/index.ts`, `hooks/index.ts`
- **One-liners** where the code is self-documenting:
  ```ts
  // BAD: comment adds nothing
  /** Returns the course ID. */
  const getId = (course: Course) => course.id;
  ```

## 2. API Service Functions

### Template

```ts
/**
 * One-sentence summary of what the function fetches or mutates.
 *
 * Optional second paragraph for side effects, constraints, or caveats.
 *

 * @returns Description of what the resolved value contains — not just the TypeScript type.
 * @throws {PostgrestError} Condition under which an error is thrown or returned.
 *
 * @example
 * ```ts

## 3. Custom Hooks

### Template

```ts
/**
 * One-sentence summary of what the hook manages.
 *
 * Key behavior: describe intervals, subscriptions, cleanup, optimistic updates.
 * Mention unmount cleanup if the hook sets up subscriptions or timers.
 *
 * @param courseId - UUID of the course to scope state to.
 * @returns Object — list each key return value and what it means in plain English.
 *
 * @example
 * ```tsx
 * function LearnPage() {
 *   const { progressPercent, checkCompletion } = useModuleProgress({
 *     courseId: 'abc-123',
 *     activeChapterId: moduleId,
 *     isPlaying: true,
 *     currentTime: videoRef.current?.currentTime ?? 0,
 *     duration: videoRef.current?.duration ?? 0,
 *   });
 * }
 * ```
 */
```

### Real example: `useModuleProgress`

```ts
/**
 * Tracks and persists video watch progress for a course module.
 *
 * Auto-saves progress to Supabase every 30 seconds while the video is playing
 * (`AUTO_SAVE_INTERVAL = 30000`). On first save of a session, increments the
 * `view_count` via the `increment_view_count` RPC. At 95% watch time
 * (`COMPLETION_THRESHOLD = 0.95`), calls the `progress-complete` Edge Function
 * to mark the module complete.
 *
 * Cleans up the auto-save interval on unmount.
 *
 * @param input - Configuration object containing courseId, activeChapterId, isPlaying,
 *   currentTime, and duration (all required for accurate tracking).
 * @returns Object with:
 *   - `progressPercent` — 0–100 overall course completion percentage
 *   - `moduleCompletionMap` — Record<moduleId, boolean> of which modules are done
 *   - `showCompletionNotification` — true briefly after a module is marked complete
 *   - `checkCompletion` — imperative trigger for completion check (used by VideoPlayer)
 */
```

### Hook lifecycle documentation pattern

For hooks that manage subscriptions or timers, always document:
1. What is set up on mount
2. The trigger condition for the subscription/interval
3. What is cleaned up on unmount

```ts
/**
 * Subscribes to real-time INSERT events on the `notifications` table for the
 * current user. The subscription is established on mount and cleaned up on unmount.
 * New notifications are prepended to the list without a full refetch.
 */
```

## 4. Component Props

### Template

```ts
/**
 * Props for the [ComponentName] component.
 *
 * [Note: any usage constraints — requires a context provider, uses forwardRef,
 * exposes an imperative handle, etc.]
 */
interface ComponentNameProps {
  /** What this prop does. Valid values or format if not obvious from the type. */
  propName: Type;
  /** What undefined means (only if non-obvious). @default [value] */
  optionalProp?: Type;
}
```

### Real example: `VideoPlayerHandle`

```ts
/**
 * Imperative handle exposed by VideoPlayer via React.forwardRef.
 *
 * Mount the VideoPlayer with a ref to access these controls from a parent component:
 * ```tsx
 * const videoRef = useRef<VideoPlayerHandle>(null);
 * videoRef.current?.play();
 * ```
 */
export interface VideoPlayerHandle {
  /** Starts video playback. No-op if already playing. */
  play: () => void;
  /** Pauses video playback. No-op if already paused. */
  pause: () => void;
  /** Seeks to the specified time in seconds. Clamps to [0, duration]. */
  seek: (time: number) => void;
  /** Returns the current playback position in seconds. Returns 0 if not mounted. */
  getCurrentTime: () => number;
  /** Returns the total video duration in seconds. Returns 0 before metadata loads. */
  getDuration: () => number;
}

/**
 * Props for the VideoPlayer component.
 *
 * Requires HLS.js to be available in the environment (not a concern in browsers;
 * see HLS.js browser support matrix for compatibility details).
 */
interface VideoPlayerProps {
  /** Bunny.net video GUID (not a URL). Used to construct the HLS playlist URL. */
  videoId: string;
  /** Database module ID — passed to useModuleProgress for progress tracking. */
  moduleId: string;
  /**
   * CDN fallback URL served immediately while the signed URL is being fetched.
   * If signed URL generation fails, this URL is used permanently for the session.
   */
  fallbackUrl?: string;
}
```

## 5. Edge Function Handlers

### Template

```ts
/**
 * [FunctionName] Edge Function — one-sentence purpose.
 *
 * Auth: JWT required | No JWT (verified by HMAC) | JWT + admin role required
 * Method: POST
 *
 * Request body:
 * ```json
 * { "field": "type description" }
 * ```
 *
 * Response (success):
 * ```json
 * { "success": true, "fieldName": "value" }
 * ```
 *
 * Side effects: [List DB writes, emails sent, external API calls]
 */
Deno.serve(async (req: Request) => {
```

### Real example: `checkout-webhook`

```ts
/**
 * checkout-webhook Edge Function — handles Razorpay async webhook for payment confirmation.
 *
 * Auth: NO JWT. This function is called by Razorpay's servers, not the frontend.
 * Identity is verified via HMAC-SHA256 signature on the `X-Razorpay-Signature` header.
 * Never add JWT verification to this function.
 *
 * Method: POST
 *
 * This is an async safety net: if the user closes the browser after payment but before
 * the `checkout-verify` callback fires, Razorpay calls this webhook within minutes,
 * ensuring the enrollment is eventually created.
 *
 * Side effects:
 * - Creates enrollment in `enrollments` table (if not already exists)
 * - Creates payment record in `payments` table
 * - Sends enrollment confirmation email via Resend
 */
```

## 6. Anti-Patterns

### Commenting what the code does (not the why)

```ts
// BAD: tells us nothing we can't see
/** Gets the user. */
async function getUser(id: string): Promise<User>

// GOOD: tells us the why and the return contract
/**
 * Fetches the authenticated user's profile from `public.users`.
 * Returns null if the profile row hasn't been created yet (race with auth trigger).
 */
async function getUser(id: string): Promise<User | null>
```

### Redundant @param types

```ts
// BAD: TypeScript already has the type
@param courseId {string} - The course ID

// GOOD: describe what the value IS, not its type
@param courseId - UUID of the course; must correspond to a published course.
```

### Missing @example for non-obvious return shapes

```ts
// BAD: return is complex; no example
@returns The module progress data.

// GOOD
@returns Object with progressPercent (0–100), moduleCompletionMap (Record<id, boolean>),
  and checkCompletion (imperative trigger). See @example for usage.
```

### JSDoc on auto-generated code

Never add JSDoc to `types/supabase.ts`. It is overwritten by `supabase gen types typescript`.

## 7. VS Code Tooling

### Hover tooltips

With JSDoc in place, hovering over a function call in VS Code shows the docstring in a
tooltip. This is the primary consumer of the JSDoc — write for the developer hovering,
not for generated HTML documentation.

### Auto-completion context

VS Code uses `@param` descriptions in parameter hints. Precise `@param` docs eliminate
the need to navigate to the source file to understand what a parameter expects.

### Recommended extensions

- **ESLint** — catches some JSDoc violations via `eslint-plugin-jsdoc` (optional setup)
- **TypeScript** — built-in JSDoc rendering in hover tooltips

### Verifying your JSDoc renders

1. Write the JSDoc comment
2. Hover over the function call site in VS Code
3. Confirm the tooltip shows your summary and parameters
4. If no tooltip: check for syntax errors in the JSDoc block (unclosed `*/`, missing `*`)
